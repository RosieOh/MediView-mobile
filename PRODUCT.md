# Product — Mobile App

루트 [`../PRODUCT.md`](../PRODUCT.md) 를 상속한다. 서비스의 **본체**이자 주력 채널.

## Register

product

## Platform

adaptive

Expo/React Native 로 iOS·Android 를 함께 出荷한다. 화면 구성은 공유하되, 네이티브 관례
(iOS HIG / Android Material 3)를 존중한다 — 내비게이션 제스처, 하단 탭, 시스템 폰트 크기 설정,
햅틱, 권한 요청 문구는 각 OS 규칙을 따른다.

## Users

- **환자** — 아픈 상태에서 접속. 한 손 조작, 불안정한 네트워크, 다양한 화면 크기.
  고령 사용자 비중이 유의미하므로 시스템 글자 크기 확대를 반드시 지원한다.
- **의료진** — 진료 사이 짧은 틈에 대기열 확인, 문진 검토, 처방·서류 발급.

## Product Purpose

**접속 → 의사 연결 → 진료 → 결과 수령(처방·서류·결제)** 의 전 과정을 앱 하나에서 끝낸다.
성공은 "이탈 없이 진료를 완료한 비율"과 "연결까지 걸린 시간".

## Brand Personality

루트와 동일(**빠름 · 명료함 · 유능함**). 앱에서는 특히 **군더더기 제거**로 드러난다.
환영 애니메이션, 온보딩 캐러셀 같은 지연 요소를 넣지 않는다.

## Anti-references

- 진료 시작 전에 끼어드는 온보딩 캐러셀·환영 모달
- 의료 정보를 가리는 장식 요소
- 화상 진료 중 화면을 뒤덮는 배너·토스트
- 어두운 진료실 배경에서 대비가 무너지는 아이콘 (수정 완료 — 되돌리지 말 것)

## Design Principles

1. **한 화면에 한 결정.** 아픈 사람이 고르게 하지 않는다. 다음 행동은 항상 하나가 눈에 띈다.
2. **네트워크를 의심한다.** 오프라인 배너, 당겨서 새로고침, 실패 시 재시도가 기본.
   화상 통화는 P2P 실패를 전제로 TURN 릴레이를 준비한다.
3. **대기 시간을 정직하게.** 남은 순번·예상 시간을 보여준다. 빈 스피너로 방치하지 않는다.
4. **한 손으로 닿는 곳에.** 주요 액션은 화면 하단, 터치 타깃 44pt 이상.
5. **의료 판단은 사람이.** AI 요약은 초안임을 명시하고, 의료진 확인 전에는 확정처럼 보이지 않게 한다.

## Accessibility & Inclusion

WCAG 2.1 AA 상당 + 고령 사용자 배려.
Dynamic Type / 시스템 글자 크기 확대 대응, 터치 타깃 44pt 이상,
상태는 색상 단독 금지, `prefers-reduced-motion`(Reduce Motion) 존중.
화상 진료는 저대역폭·불안정 네트워크를 기본 전제로 한다.
